# required属性(HTML5バリデーション)

HTML5の`<input>`, `<select>`, `<textarea>`要素に追加できるブール属性(値は不要)。

フォームが送信される前に、そのフィールドが空でないことを保証する。

ブラウザ側で処理するので、サーバーへの通信は発生しない。

ブラウザが定義した標準のメッセージ(例:Chromeなら「このフィールドを入力してください」)を吹き出しで表示する。

## Laravelのバリデーションとの優先順位

クライアントサイド(HTML5) => サーバーサイド(Laravel)の順。

- ブラウザ側で最も高速なフィードバック、サーバー通信を避ける
- サーバー側ではセキュリティを確保、クライアント側で無効化されたチェックを補完

=> Laravelのバリデーションをテストするときには、一時的に無効化にする必要がある

=> ただし、実運用時はブラウザの利便性を活かすためにも両方設定することが推奨される

## HTML5の主な入力チェック属性

### 必須チェック(required)

- 属性: `required`
- フィールドが空欄の場合送信を阻止(最も基本的なチェック)
- 例: `<input type="text" required>`

### 型チェック(type属性)

`type`属性を変えるだけで、ブラウザが自動的にその型のフォーマットをチェックし、標準エラーメッセージを表示。

| type     | 役割                  | ブラウザの挙動                         |
|----------|-----------------------|----------------------------------------|
| `email`  | Eメール形式かチェック | `@`と`.`が含まれているか               |
| `url`    | URL形式かチェック     | `https://`やドメイン名が含まれているか |
| `number` | 数値のみ許可          | 数字以外が入力されていないか           |

### 範囲・長さのチェック(min, max, maxlength)

特に`type="number"`と組み合わせることで、数値の範囲チェックを簡単に行える。

| 属性        | 役割                                 | 例・補足                                 |
|-------------|--------------------------------------|------------------------------------------|
| `min`       | 数値の最小値、日付の最小日           | `<input type="number" min="0">`          |
| `max`       | 数値の最大値、日付の最大日           | `<input type="number" max="999">`        |
| `step`      | 数値の増減幅                         | 入力できる数位がこの倍数であることを強制 |
| `maxlength` | 入力できる文字数の最大値             | これ以上の文字を入力できなくなる         |
| `minlength` | 入力しなければならない文字数の最小値 | 設定した文字数に満たない場合エラー       |

### より複雑なチェック(pattern)

`pattern`属性を使用すれば正規表現を使った形式チェックが可能。

電話番号(ハイフンありなし)や特定の英数字の組み合わせなど、複雑なルールを強制できる。

例: `<input pattern="[0-9]{3}-[0-9]{4}">` => `数字3桁` `-(ハイフン)` `数字4桁`の形式

## 注意点

HTMLのチェックは簡単に回避できてしまうため、サーバーサイドでのバリデーションチェックは必ず必要。

また、HTMLチェックでは実現できない複雑なルールなども必ずサーバー側のバリデーションで実施する。
