# MVC

LaravelはModel-View-Controller(MVC)という設計パターンを採用している。

アプリケーションの各部分に明確な役割を与え、コードを整理するための仕組み。

アプリケーションへのリクエストは以下の流れで処理される。

| 要素          | 役割                 | ディレクトリ(主要な格納場所) |
|---------------|----------------------|------------------------------|
| Controller(C) | 交通整理係、司令塔   | `app/Http/Controllers`       |
| Model(M)      | データ処理係、DB連携 | `app/Models`                 |
| View(V)       | 表示係、画面         | `resources/views`            |

## リクエストの流れ(一例)

1. Routeing: ユーザーが特定のURL(例: `/users`)にアクセスすると、どのControllerのどのメソッドを呼び出すかを決定する。

2. Controller: 処理を受け取り、Modelにユーザー一覧のデータを取得するように指示。

3. Model: データベースと通信し、必要なデータを取得。

4. Controller: Modelからデータを受け取り、Viewにこのデータを埋め込んでHTMLを生成するように指示。

5. View: データを受け取り、最終的なHTMLを生成し、ブラウザに返す。

## プロジェクトの主要ディレクトリ

| ディレクトリ名 | 役割                                                           |
|----------------|----------------------------------------------------------------|
| `app/`         | Model, Controller, ServiceなどのPHPロジックを格納              |
| `resources/`   | 画面に関するファイル(Viewテンプレート、CSS、JSなど)を格納      |
| `routes/`      | ルーティング定義ファイルを格納                                 |
| `database/`    | DBのマイグレーション(テーブル定義)やシーダー(初期データ)を格納 |
| `config`       | アプリケーション全体の各種設定ファイル                         |
