# Sizing

Tailwindでは`w-*`で幅(width)、`h-*`で高さ(height)を制御する。

## 相対的な幅の制御

要素の幅を親要素や画面(ビューポート)のサイズを基準に設定するときに使う。

| Tailwind     | CSS相当           | 役割                                                       |
|--------------|-------------------|------------------------------------------------------------|
| `w-full`     | `width: 100%`     | 親要素のいっぱいに広がる                                   |
| `w-1/2`      | `width: 50%`      | 親要素の幅の半分(`w-1/3`, `w-2/5`なども可能                |
| `min-w-full` | `min-width: 100%` | 最小限の幅を親要素の幅いっぱいに設定                       |
| `w-screen`   | `width: 100%vw`   | 画面全体の幅いっぱいに広がる(親要素ではなく画面全体が基準) |

## 固定的な幅の制御

ピクセル値に基づいた固定幅を指定するときに使用。

Tailwindでは、4倍の数を基本とした独自のスケール(間隔)を採用している。

| Tailwind | CSS相当        | Pixel |
|----------|----------------|-------|
| `w-4`    | `width: 1rem`  | 16px  |
| `w-8`    | `width: 2rem`  | 32px  |
| `w-16`   | `width: 4rem`  | 64px  |
| `w-64`   | `width: 16rem` | 256px |

## max-w- 系の使い方

レスポンシブデザインの品質を大きく左右する重要なクラス。

| Tailwind | CSS相当 | 役割 |
|----------|---------|------|
| `max-w-full`      | `max-width: 100%;`   | 要素の幅が親要素の幅を超えないようにする(安全策) |
| `max-w-lg`        | `max-width: 32rem;`  | 要素の幅が最大でも32rem(512px)                   |
| `max-w-screen-x1` | `max-width: 1280px;` | 画面幅が広くても、コンテンツの幅を1280pxに制限   |

### なぜ w-full と max-w- を一緒に使うのか？

多くのウェブサイトのコンテンツエリアは`<div class="w-full max-w-lg"> ...`で成り立っている。

- モバイル時: 画面幅が狭い時は、`w-full`が効き、要素は画面いっぱいに広がる
- デスクトップ時: 画面幅が広いときは、`max-w-lg`が効き、要素は最大幅(例: 512px)に制限

もし`max-w-lg`がなかったら、巨大なデスクトップ画面いっぱいにフォームが広がり、入力しにくい間延びしたデザインになってしまう。

## w-auto

CSSでは`width: auto`に相当。

要素の幅をブラウザのデフォルトの動作に任せる。

`width: auto`は要素の`display`プロパティ(ブロック要素かインライン要素)によって振る舞いが変わる。

### プロック要素の場合

ブロック要素のデフォルトの幅は`width: 100%(w-full)`なので、`w-auto`を指定してもほとんどの場合は親要素の幅いっぱいに広がる。

ただし、マージンによる中央寄せ(`mx-auto`)を使用する場合は、`w-auto`が暗黙的に必要となる。

```
<div class="w-96" mx-auto> <!-- 幅を指定し、mx-autoで中央寄せ -->
```

内部的には`width: auto`が有効になっていなくても、`mx-auto`が機能するためには幅が100%未満である必要がある。

`w-*`で具体的な幅を指定すると、自動的に`w-auto`の性質が活かされる。

### インライン / インラインブロック要素

インライン要素のデフォルトの幅は、中身のコンテンツの幅(`fit-content`)

- `w-full`: インライン要素でも強制的に親要素の幅いっぱいに広がる
- `w-auto`: 強制的に広げられた幅をリセットし、元のコンテンツの幅に戻る

Flexbox内で特定の愛アテムに一時的に`w-full`を適用した後、別のブレークポイントで元のサイズに戻したいときに`w-auto`が使われる。

```
<div class="flex">
  <!-- モバイル(デフォルト)では幅いっぱい -->
  <div class="w-full sm:w-auto">
  <!-- スモールスクリーン以上ではコンテンツの幅に戻る -->
  ...
  </div>
</div>
```
